Two Technical Approaches

01

Usage Pattern Mining

ECOOP (2009)

  • Mines large repositories to detect common method call sequences automatically.

  • Leverages frequent usage patterns ("wisdom of the crowds") to guide correct API updates.

Caveat

Frequent ≠ correct. Results depend heavily on whose code is being mined (bias propagation).

02

Doc-Based Matching

ACM SIGSOFT ISSTA 2018

  • Converts documentation into formal specifications and machine-readable constraints.

  • Enables Natural Language → Code mapping to find semantically similar API replacements.

Caveat

Documentation quality varies widely, making this approach fragile for poorly documented libraries.
